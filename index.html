Distinguish between TCP and UDP

The Internet’s transport layer transports application-layer messages between application endpoints. In the Internet there are two transport protocols, TCP and UDP, either of which can transport application- layer messages.

What is TCP (Transmission Control Protocol)?

TCP provides a ­connection-oriented service to its applications. This service includes guaranteed delivery of application-layer messages to the destination and flow control (that is, sender/receiver speed matching). TCP also breaks long messages into shorter ­segments and provides a congestion-control mechanism, so that a source throttles its transmission rate when the network is congested.

Transmission Control Protocol (TCP) is a connection-oriented protocol that computers use to communicate over the internet. It is one of the main protocols in TCP/IP networks. TCP provides error-checking and guarantees delivery of data and that packets will be delivered in the order they were sent.

What is UDP (User Datagram Protocol)?

The UDP protocol provides a connectionless service to its applications. This is a no-frills service that provides no reliability, no flow control, and no congestion control. In this book, we’ll refer to a transport-layer packet as a segment.

User Datagram Protocol (UDP) is a connectionless protocol that works just like TCP but assumes that error-checking and recovery services are not required. Instead, UDP continuously sends datagrams to the recipient whether they receive them or not.

The difference between TCP and UDP

TCP and UDP have many differences and similarities. They are the most commonly used protocols for sending packets over the internet. They both work on the transport layer of the TCP/IP protocol stack and both use the IP protocol.

Let’s take a look at some of the key differences.

Connection and connectionless

TCP is a connection-oriented protocol and UDP is a connection-less protocol. TCP establishes a connection between a sender and receiver before data can be sent. UDP does not establish a connection before sending data.

Reliability

TCP is reliable. Data sent using a TCP protocol is guaranteed to be delivered to the receiver. If data is lost in transit it will recover the data and resend it. TCP will also check packets for errors and track packets so that data is not lost or corrupted.

UDP is unreliable, it does not provide guaranteed delivery and a datagram packet may become corrupt or lost in transit.

Flow control

TCP uses a flow control mechanism that ensures a sender is not overwhelming a receiver by sending too many packets at once. TCP stores data in a send buffer and receives data in a receive buffer. When an application is ready, it will read the data from the receive buffer. If the receive buffer is full, the receiver would not be able to handle more data and would drop it. To maintain the amount of data that can be sent to a receiver, the receiver tells the sender how much spare room in the receive buffer there is (receive window). Every time a packet is received, a message is sent to the sender with the value of the current receive window.

UDP does not provide flow control. With UDP, packets arrive in a continuous stream or they are dropped.

Ordering

TCP does ordering and sequencing to guarantee that packets sent from a server will be delivered to the client in the same order they were sent. On the other hand, UDP sends packets in any order.

Speed

TCP is slower than UDP because it has a lot more to do. TCP has to establish a connection, error-check, and guarantee that files are received in the order they were sent.

Usage

TCP is best suited to be used for applications that require high reliability where timing is less of a concern.

World Wide Web (HTTP, HTTPS)
Secure Shell (SSH)
File Transfer Protocol (FTP)
Email (SMTP, IMAP/POP)
UDP is best suited for applications that require speed and efficiency.

VPN tunneling
Streaming videos
Online games
Live broadcasts
Domain Name System (DNS)
Voice over Internet Protocol (VoIP)
Trivial File Transfer Protocol (TFTP)

The Overview of FR and NFR

2. Functional and non-functional requirements

Notice: Language for Writing Requirements

  • MoSCoW Criteria :
  • M : Must have-mandatory requirements that are fundamental to the system
  • S : Should have –important requirements that could be omitted
  • C : Could have –optional requirements
  • W : Want to have –these requirements really can wait (i.e. bells and whistles)

2.1 Functional requirements

Adult Users

  1. The system must allow viewers to create an account by using the username and password and choose their account type.
  2. The system must provide services for three types of account, the adult users, viewers and business.
  3. The system must allow all adult users, viewers, businesses and retailers to sign in by their username and password.
  4. The system must allow all user to reset their password and change the detail of their account.
  5. The system must allow adult users to post videos
  6. The system must allow adult users to have a video station.
  7. The system must allow adult users to watch, comment, search different videos and follow the station
  8. The system must only allow registered users to manage individual accounts.
  9. The system shall allow adult users to receive videos recommended by users’ preference.

Viewers

  1. The system must allow viewers to watch different videos and search for videos by name and author.
  2. The system must allow viewers to watch the advertisements.
  3. The system shall allow viewers to search, comment, rank advertisements and their providers.
  4. The system must allow viewers to skip adverts.
  5. The system could push videos according to users’ preferences.
  6. The system must permit users to access videos from the other companies.

Business and Retailers

  1. The system must allow businesses to post some adverts and broadcasts.
  2. The system must allow businesses to manage their adverts and broadcast.

2.2 Non-functional requirements

Security

  • The system must protect users’ privacy including personal information, preference and location and language.
  • The system should check abnormal accounts by some smart mechanisms, and then instantly lock it up after potential dangers are found.

Feasibility

  • The system could recommend relevant videos based on their usage, such as personal profile, preference, use frequency, geographic location, language, lifestyle.
  • The system should allow paid users to skip advertisements.

Extensibility

  • The system shall meet those requirements with the increasing users

Availability

  • The system shall be available 24/7 by some reasonable methods like the redundant technique.

Validity

  • The system must automatically check the validity of the contents and provide manual ways to deal with special situations.
  • The system must provide effective ways using gamification techniques to suffer from digital addiction.

2.3 The combination of FR and NFR

Security

  • All user data must be multi-encrypted.
  • The use of user data by the Videosharetube platform must follow the mutually agreed privacy terms.
    Availability:
  • The system must allow ordinary viewers (unregistered users) to access the Videosharetube website as well.

Reliability

  • The system must allow adults users to have multiple payment methods for payment orders, including credit/debit cards or via direct debit.
  • The third-party financial institutions must ensure that users’ payments are secure.

Usability

  • The system will periodically push advertisements and promotional offers to different platforms, including phones, emails and screens.

The analysis of the project

Scope of the project

1 System description

The aim of the project is to engineer a video-related software system named VideoSharetube to be used to share videos from various sources. Our project aims to provide a perfect application environment for different groups with varying purposes, including entertainment, advertisement and promotions. In order to meet as many user’s requirements as possible, it is considered that there will be types of users as follows, according to the analysis of relevantly existing applications.

Users

For adult users, they can sign up to Videosharetube to create profiles and post videos as well as discuss, share and comment on videos. It is significant for those users who are enthusiastic about sharing videos to have a station and follow the stations. And personal information is also allowed to be customized to protect their privacy as well as possible.

For general viewers, skipping and viewing advertisements are fundamental functions. If users are interested in advertisements, they can search for advertisements that they want. With regard to those advertisements searched, the advertisements and their providers are more likely to be rated based on comments and marks from users, which would provide a valuable reference for other users. In addition, users have to be charged above £1 per hour while accessing to online videos provided by other video companies.

Meanwhile, Videosharetube not only provides services for personal users, but also business users are essential clients of Videosharetube. Some advertising companies or enterprises (such as electrical appliances retailers) who need to increase their popularity quickly can subscribe to Videosharetube for advertising and broadcasting to raise their brand awareness and product awareness. They need to pay £300 to Videosharetube to join the Videosharetube branding service, and then pay £500 per month depending on the duration of the service.

The payment system is also indispensable. When subscriptions and payments take place on the Videosharetube platform, the third party can be used as an intermediary to manage and pay the corresponding transitions. There are multiple payment ways supported, such as credit cards, debit cards and direct debit.

When it comes to the application itself, there are some functions as follows. One is the profile subsystem where people can upload and manage their personal information which would be used by the platform to recommend videos and send advertisements to people through phones, screens and emails.

In terms of security, Videosharetube would try their best to protect users’ privacy using updating encryption method or deleting leaked data, once it is found that the personal information could be used illegally like maliciously leaked.

Moreover, Videosharetube will also monitor people’s behaviours. The first way to prevent digital addiction is to use gamification techniques which would award various badges rely on different usage time. Besides, Videosharetube review all materials, videos, and posts uploaded by users in accordance with laws and company terms automatically. If these do not pass automated review, relevant platform staffs will moderate it again.

More to the point is reliability and availability. With the increasing number of online users, Videosharetube will keep adjusting the above functions to satisfy different users’ requirements.

Overall, Videosharetube will find the most suitable method from security, availability, reliability, and real-time performance to obtain satisfactory results.

What is Unified Modeling Language (UML)?

What UML

UML, short for Unified Modeling Language, is a standardized modeling language consisting of an integrated set of diagrams, developed to help system and software developers for specifying, visualizing, constructing, and documenting the artifacts of software systems, as well as for business modeling and other non-software systems. The UML represents a collection of best engineering practices that have proven successful in the modeling of large and complex systems. The UML is a very important part of developing object oriented software and the software development process. The UML uses mostly graphical notations to express the design of software projects. Using the UML helps project teams communicate, explore potential designs, and validate the architectural design of the software. In this article, we will give you detailed ideas about what is UML, the history of UML and a description of each UML diagram type, along with UML examples.

Why UML

As the strategic value of software increases for many companies, the industry looks for techniques to automate the production of software and to improve quality and reduce cost and time-to-market. These techniques include component technology, visual programming, patterns and frameworks. Businesses also seek techniques to manage the complexity of systems as they increase in scope and scale. In particular, they recognize the need to solve recurring architectural problems, such as physical distribution, concurrency, replication, security, load balancing and fault tolerance. Additionally, the development for the World Wide Web, while making some things simpler, has exacerbated these architectural problems. The Unified Modeling Language (UML) was designed to respond to these needs. The primary goals in the design of the UML summarize by Page-Jones in Fundamental Object-Oriented Design in UML as follows:

  1. Provide users with a ready-to-use, expressive visual modeling language so they can develop and exchange meaningful models.
  2. Provide extensibility and specialization mechanisms to extend the core concepts.
  3. Be independent of particular programming languages and development processes.
  4. Provide a formal basis for understanding the modeling language.
  5. Encourage the growth of the OO tools market.
  6. Support higher-level development concepts such as collaborations, frameworks, patterns and components.
  7. Integrate best practices.

下面我介绍的这种方法可以说是屡试不爽。我看过几乎全部的装机教程，确实每种方案都是正确的，但有时成功率让人发指。经常出现各类失败问题，在这里我就不再赘述了。

准备工具

  1. win7官方原版镜像文件（cn_windows_7_ultimate_with_sp1_x64_dvd_u_677408.iso），推荐到MSDN,我告诉你下载。
  2. 老毛桃u盘制作工具
  3. 一个8G以上的U盘

装机步骤

  1. 用老毛桃U盘制作工具制作U盘启动盘（自行教程，太简单）。
  2. 将win7官方镜像文件用解压软件解压。
  3. 把这三个文件复制到U盘boot(文件夹)，sources（文件夹），bootmgr（文件）。
  4. 此时关机，插入做好的U盘，按F8（不同电脑有差别，自行解决），选择U盘启动。
  5. 进入系统后，用mini系统自带磁盘工具格式化C盘（系统盘）
  6. 将上述三个文件复制到C盘（系统盘）。
  7. 按在Win+R调出运行窗口，输入c:\boot\bootsect.exe /nt60 c:，回车。
  8. 看到黑色窗口一闪而过，如果反应够快最后一行可以看到successful这个词，说明已经成功。
  9. 此时关机，拔掉U盘后，再次开机，进入熟悉的Windows装系统界面。
