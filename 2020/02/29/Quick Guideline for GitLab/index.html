Quick Guideline for GitLab

发表于 2020-02-29 | 分类于 JavaProject | 0 comments | Hits
字数统计 444 | 阅读时长 2

Notice : You won’t be able to pull or push project code via SSH until you add an SSH key to your profile
Suggest : prior to use ED25519 SSH keys, since they are more secure and have better performance over the other types.
About adding an SSH key See Offical Details

Config SSH key

Take the following steps:

  1. Open terminal

  2. Generate a new ED25519 SSH key pair:
    ssh-keygen -t ed25519 -C "username@cs.bham.ac.uk"

  3. go to the path ~/.ssh/ to find the file id_ed25519.pub and use any editor to open it.

    The .shh is a hidden filefolder, it can be opened after getting visiable.

  4. Add your public SSH key to your GitLab account by:

Clicking your avatar in the upper right corner and selecting Settings.
Navigating to SSH Keys and pasting your public key from id_ed25519.pub into the Key field.

  1. Click the Add key button.

Common Git Instruction

Due to the GitLab Repository already established, we are supposed to bagin with the clone.

  1. Install Git
    if not done yet, Please install git based on your corresponding OS

  2. Open terminal and Identify yourself to Git by setting:

  3. Clone your own Project by your repository address.
    git clone yourProjectAddress

    Like https://git-teaching.cs.bham.ac.uk/mod-msc-sw1-2019/melbourne.git

  4. Create new branch
    git branch yourBranchName

  5. Switch branch
    git checkout branchName

  6. Delete branch
    git branch -d branchName
    git branch -D branchName

  7. Stage changes
    git add

  8. Commit changes
    git commit -a -m 'yourCommitText'

  9. Look at changes

  10. Merge
    git merge anotherBranchName

    merges changes from branch “name” into active branch

  11. Downloads and merges changes from the remote repository.
    git pull

  12. Sends local changes to remote repository.
    git push

  13. Downloads changes but does not merge them. (Merge manually later.)
    git fetch

Using the School’s VPN service

Connection details :

Via: Computer Science Support Pages: VPN

  1. Open System Preferences -> Network -> Add -> Interface: VPN -> Type: L2TP -> Create ->|
  2. Server Address: vpn2.cs.bham.ac.uk
  3. Account Name: yourStudentAccount
  4. Click Authetication
  5. Password: yourStudentPasswd
  6. Machine Authetication -> Shared Secret: woolybronzeicebox

    This shared secret may change from time to time - the latest secret will always be available on this page.
